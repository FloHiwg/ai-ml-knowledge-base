## Overview

"Distillation" appears in three distinct, non-overlapping contexts in the ML literature. They share the core idea of transferring knowledge from one model (or representation) to another, but differ in what is being transferred and how.

---

## 1. Knowledge Distillation (Hinton et al. 2015)

**What it is:** Training a smaller *student* model to mimic the output distribution of a larger *teacher* model, rather than training from scratch on hard labels.

**How it works:**

```
Hard label loss:    CE(student_logits, ground_truth)
Distillation loss:  KL(student_softmax, teacher_softmax) × temperature²
Total loss:         α × hard_label + (1-α) × distillation
```

Soft labels from the teacher (e.g., `[0.85 cat, 0.10 dog, 0.05 bird]`) carry richer information than hard labels (`[1, 0, 0]`) — they encode the teacher's uncertainty and the relationships between classes.

**Used in continual learning:** A distillation loss that penalizes the student for drifting from the *prior model's* outputs during new-task training. Keeps representations anchored to prior behavior without needing old training data. See [[training/continual-learning]].

---

## 2. DeiT Distillation Token (Touvron et al. 2021)

**What it is:** A ViT-specific technique where a CNN teacher's predictions are used to train a dedicated *distillation token* in the Vision Transformer.

**How it works:**

Three input tokens: `[CLS]`, patch tokens, and a `[distillation]` token. The distillation token's final state is trained to match the CNN teacher's output (hard or soft). The CLS token is trained on true labels as normal.

```
CLS loss:           CE(cls_output, ground_truth_label)
Distillation loss:  CE(distillation_output, teacher_argmax)   # hard distillation
```

At inference, both the CLS and distillation token outputs are fused for the final prediction.

**Why it works:** The CNN teacher has built-in spatial inductive biases (locality, translation equivariance) that ViT must learn from data. The distillation token specifically absorbs these biases, allowing ViT-level accuracy on ImageNet without the massive external pretraining (JFT-300M) that standard ViT requires.

See [[architecture/vision-transformers]].

## 3. Capability Distillation (DeepSeek-R1, 2025)

**What it is:** Training a small base model via SFT on outputs generated by a large, capable model — transferring reasoning *patterns* rather than just output distributions.

**How it works:**

```
1. Large model (e.g., DeepSeek-R1 671B) generates high-quality reasoning trajectories
2. Filter for correctness and quality
3. Fine-tune small base model (e.g., Qwen-2.5-7B, LLaMA-3-8B) on this dataset via standard SFT
```

No RL is run on the small model — it just learns from the large model's demonstrated reasoning.

**Key results:**
- Distilled Qwen2.5-14B > QwQ-32B-Preview (prior best open reasoning model)
- Distilled 32B/70B > o1-mini on most benchmarks
- **Distillation > direct RL on small models**: reasoning patterns from large models transfer more efficiently than re-learning them from scratch via RL

**Why it works:** RL on large models discovers the *structure* of good reasoning (when to backtrack, how to decompose, when to self-check). SFT on a small model can absorb this structure cheaply from demonstrated examples, even though the small model could never discover it via RL alone.

See [[training/reasoning-models]].

## Comparison

| | Knowledge Distillation | DeiT Distillation Token | Capability Distillation |
|---|---|---|---|
| **Teacher** | Any large model | Pretrained CNN | Large reasoning model (R1) |
| **Student** | Smaller model, same task | ViT (different architecture) | Small base LLM |
| **What transfers** | Output distribution (soft labels) | Spatial inductive bias | Reasoning patterns (trajectories) |
| **Training signal** | KL divergence from teacher logits | Cross-entropy vs teacher argmax | Standard SFT on generated data |
| **Goal** | Efficiency — same task, smaller model | Data efficiency — ViT without JFT | Capability — reasoning without RL cost |
| **Context** | General ML; continual learning regularization | Vision transformers | LLM reasoning models |
